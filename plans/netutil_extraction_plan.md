# План: Вынос функции getLocalIP в общий пакет

## Задача
Согласно замечанию из ревью, функция `getLocalIP` нарушает принцип DRY (Don't Repeat Yourself). Необходимо вынести её в общий пакет для повторного использования.

## Текущее состояние
Функция `getLocalIP` дублируется в двух местах:

| Файл | Строки | Тип |
|------|--------|-----|
| `internal/grpc/client.go` | 155-200 | Обычная функция |
| `internal/agent/agent.go` | 152-198 | Метод структуры Agent |

Обе функции практически идентичны по реализации и логике.

## Использование функции

### В internal/grpc/client.go:
- Вызывается в методе [`NewClient()`](internal/grpc/client.go:45)
- IP сохраняется в поле `localIP` структуры `Client`
- Используется для добавления в метаданные gRPC запросов

### В internal/agent/agent.go:
- Вызывается в методе [`makeRequest()`](internal/agent/agent.go:372)
- IP добавляется в заголовок HTTP запроса `X-Real-IP`
- Не использует поля структуры Agent (приёмник не нужен)

## План действий

### 1. Создать новый пакет `internal/netutil`
- Создать директорию `internal/netutil`
- Создать файл `netutil.go` с экспортируемой функцией `GetLocalIP()`

### 2. Реализовать функцию в новом пакете

Функция должна:
- Перебирать сетевые интерфейсы
- Фильтровать отключённые интерфейсы (net.FlagUp)
- Фильтровать loopback интерфейсы (net.FlagLoopback)
- Возвращать первый найденный IPv4 адрес

Сигнатура:
```go
func GetLocalIP() (string, error)
```

### 3. Рефакторинг internal/grpc/client.go
- Удалить локальную функцию `getLocalIP()`
- Добавить импорт `"github.com/paxren/metrics/internal/netutil"`
- Заменить вызов `getLocalIP()` на `netutil.GetLocalIP()`

### 4. Рефакторинг internal/agent/agent.go
- Удалить метод `getLocalIP()` структуры `Agent`
- Добавить импорт `"github.com/paxren/metrics/internal/netutil"`
- Заменить вызов `a.getLocalIP()` на `netutil.GetLocalIP()`

### 5. Верификация
- Проверить, что код собирается без ошибок
- Запустить тесты для проверки сохранения функциональности

## Преимущества рефакторинга
- **Устранение DRY violation**: Дублирование кода удалено
- **Повторное использование**: Функция может использоваться другими компонентами
- **Отделение ответственности**: Сетевые утилиты не привязаны к конкретным пакетам
- **Тестируемость**: Легче писать изолированные тесты для пакета netutil

## Зависимости
- Требуется пакет standard library `net`
- Не требуется новых внешних зависимостей

## Примечания
- Функция делается экспортируемой (GetLocalIP) для использования в других пакетах
- Метод Agent.getLocalIP() не использует поля структуры, поэтому приёмник не нужен
