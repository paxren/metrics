# go-musthave-metrics-tpl

Шаблон репозитория для трека «Сервер сбора метрик и алертинга».

## Начало работы

1. Склонируйте репозиторий в любую подходящую директорию на вашем компьютере.
2. В корне репозитория выполните команду `go mod init <name>` (где `<name>` — адрес вашего репозитория на GitHub без префикса `https://`) для создания модуля.

## Обновление шаблона

Чтобы иметь возможность получать обновления автотестов и других частей шаблона, выполните команду:

```
git remote add -m main template https://github.com/Yandex-Practicum/go-musthave-metrics-tpl.git
```

Для обновления кода автотестов выполните команду:

```
git fetch template && git checkout template/main .github
```

Затем добавьте полученные изменения в свой репозиторий.

## Запуск автотестов

Для успешного запуска автотестов называйте ветки `iter<number>`, где `<number>` — порядковый номер инкремента. Например, в ветке с названием `iter4` запустятся автотесты для инкрементов с первого по четвёртый.

При мёрже ветки с инкрементом в основную ветку `main` будут запускаться все автотесты.

Подробнее про локальный и автоматический запуск читайте в [README автотестов](https://github.com/Yandex-Practicum/go-autotests).

## Проверка доверенной подсети (Trusted Subnet)

Сервер поддерживает проверку IP-адреса агента на принадлежность к доверенной подсети.

### Конфигурация

Параметр `trusted_subnet` можно задать следующими способами:

1. **Флаг командной строки:**
   ```bash
   ./server -t 192.168.1.0/24
   ```

2. **Переменная окружения:**
   ```bash
   export TRUSTED_SUBNET=192.168.1.0/24
   ./server
   ```

3. **Файл конфигурации:**
   ```json
   {
     "trusted_subnet": "192.168.1.0/24"
   }
   ```

### Формат

Значение `trusted_subnet` должно быть в формате CIDR (Classless Inter-Domain Routing), например:
- `192.168.1.0/24` - подсеть с маской /24
- `10.0.0.0/8` - подсеть с маской /8
- `172.16.0.0/16` - подсеть с маской /16

### Поведение

- Если `trusted_subnet` не задан (пустое значение), проверка отключена и все запросы пропускаются без ограничений.
- Если `trusted_subnet` задан, сервер проверяет IP-адрес из заголовка `X-Real-IP` каждого запроса.
- Если IP-адрес не входит в доверенную подсеть, сервер возвращает статус `403 Forbidden`.
- Агент автоматически добавляет заголовок `X-Real-IP` с локальным IP-адресом машины.

### Примеры

**Запуск сервера с доверенной подсетью:**
```bash
./server -t 192.168.1.0/24
```

**Запуск агента (автоматически добавляет X-Real-IP):**
```bash
./agent -a localhost:8080
```

**Запуск сервера без проверки доверенной подсети:**
```bash
./server
```
